# done

В этой папке храним только результаты выполненных шагов.

Правило:
- не писать здесь постановки задач;
- писать только фактически сделанное;
- фиксировать изменения по файлам/подсистемам;
- добавлять краткую проверку результата (build/run/log).

Рекомендуемый формат файла шага:
- цель (в контексте выполненного шага);
- что реализовано;
- какие файлы добавлены/изменены;
- как проверено;
- итог и оставшийся техдолг (если есть).
